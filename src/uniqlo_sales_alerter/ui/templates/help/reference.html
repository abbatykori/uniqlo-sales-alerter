REST endpoints

  • /api/v1/filters  — CRUD; /{id}/snooze?duration=…, /{id}/resume, /{id}/duplicate
  • /api/v1/parsers/invoice  — POST JSON {"text"}, returns categorised sizes
  • /api/v1/sales, /api/v1/sales/check, /api/v1/products/{id}
  • /api/v1/config (PUT)  — legacy single-filter config; ignored by the matcher post step 8
  • /actions/{ignore,watch,unwatch,snooze}  — HMAC-signed action URLs
  • /health  — scheduler / DB / freshness

Config schema

See config.yaml in the repo. Key keys: store_country, ui_language, deep_discount_threshold, notifications.apprise_urls. Env-var overrides for all: see README.
